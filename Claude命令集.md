## 📝 内容创建类

### 1. 创建每日笔记
```
请为我创建今天的每日笔记，文件名格式为 YYYY-MM-DD.md，放入 01_Daily/ 目录。使用日报模板，包含以下部分：
- 今日完成事项
- 明日计划
- 灵感记录
- 待办事项
添加标签：#daily #复盘 #待办
```

### 2. 记录灵感/想法
```
我有一个新想法：[在此输入想法内容]
请将这个灵感记录到 00_Raw_Sources/notes/ 目录，创建合适的文件名，添加标签 #status/待处理 #灵感
```

### 3. 创建读书笔记
```
我正在阅读《书名》，作者是作者名
请帮我创建读书笔记，放入 02_Reading/ 目录，文件名为"书名-作者.md"
使用读书笔记模板，包含：
- 书籍基本信息
- 精华摘录
- 个人思考
- 核心观点
添加标签：#阅读 #书籍 #摘录 #笔记
```

### 4. 创建技术笔记
```
我需要记录关于[技术主题]的知识点：[在此输入内容]
请帮我创建技术笔记，放入 03_Knowledge/ 对应领域目录
按照原子化笔记原则，一篇笔记只阐述一个核心概念
添加合适的标签和内部链接
```

### 5. 创建项目文档
```
我要创建一个新项目，项目名称：[项目名称]
请在 04_Projects/ 目录下创建项目文件夹和项目MOC文档
包含：项目目标、关键里程碑、相关文档链接
添加标签：#进行中 #开发项目 #[项目名称]
```

## 🔄 内容整理类

### 6. 整理 Raw_Sources
```
请帮我整理 00_Raw_Sources/ 目录中的所有内容：
1. 阅读每个文件，提炼核心观点
2. 将其重写为原子化笔记（一篇笔记只阐述一个核心概念）
3. 根据内容移动到 03_Knowledge/ 下正确的子文件夹
4. 为每篇笔记添加 2-3 个精准的标签
5. 为笔记添加至少 3 个相关的双向链接 [[相关笔记标题]]
6. 清理原始文件
```

### 7. 批量重命名文件
```
请帮我检查并重命名 03_Knowledge/ 目录下所有不规范的文件名：
- 移除"未命名"、"新建文档"等不规范名称
- 使用清晰、描述性的中文文件名
- 保持文件扩展名为 .md
```

### 8. 添加标签到文件
```
请为以下文件添加合适的标签：
[文件路径1]
[文件路径2]
...
根据文件内容和所属领域，添加 3-5 个标签，遵循标签扁平化原则
```

### 9. 建立知识关联
```
请帮我为 [文件名] 建立内部链接：
1. 分析该文件的主题和关键词
2. 在知识库中搜索相关的笔记
3. 为至少 3 个相关笔记建立双向链接 [[相关笔记标题]]
4. 在相关笔记中也添加回链
```

## 🔍 知识分析类

### 10. 生成知识网络报告
```
请对我的知识库进行一次"体检"，生成报告包含：
1. **孤立笔记清单**：哪些笔记没有入链或出链
2. **核心枢纽节点**：哪些笔记被链接次数最多
3. **标签使用统计**：哪些标签过度使用或很少使用，给出优化建议
4. **知识缺口分析**：根据现有网络，推测哪些关联领域可能缺少内容
5. **各主题笔记数量**：统计 03_Knowledge/ 下各子目录的笔记数量
```

### 11. 搜索特定主题
```
请在知识库中搜索关于 [主题/关键词] 的所有相关笔记：
1. 列出所有相关文件的路径
2. 简要总结每个文件的核心内容
3. 生成该主题的知识地图，展示笔记之间的关联
```

### 12. 分析知识结构
```
请分析 03_Knowledge/ 目录下的知识结构：
1. 列出所有主题领域
2. 统计每个领域的笔记数量
3. 识别知识密度最高的领域
4. 找出可能需要补充内容的领域
5. 评估知识组织的合理性
```

## 📊 知识输出类

### 13. 生成主题总结
```
请帮我总结 [主题名] 领域的所有知识：
1. 收集该主题下的所有笔记
2. 提炼核心概念和关键知识点
3. 生成结构化的主题总结
4. 标注知识来源和关联笔记
```

### 14. 起草文章大纲
```
我要写一篇关于 [主题] 的文章，请帮我：
1. 从知识库中提取相关素材
2. 生成文章大纲
3. 为每个大纲部分提供可引用的知识点
4. 建议合适的文章结构
```

### 15. 制作演讲 PPT 要点
```
我要做一个关于 [主题] 的演讲，请帮我：
1. 从知识库中收集相关内容
2. 生成演讲 PPT 的要点清单
3. 按照"问题-分析-解决"的逻辑组织内容
4. 为每个要点提供可引用的知识来源
```

### 16. 生成思维导图数据
```
请为 [主题] 生成思维导图的数据结构：
1. 中心主题
2. 一级分支（主要概念）
3. 二级分支（细分概念）
4. 三级分支（具体知识点）
5. 标注每个节点的关联笔记
```

### 17. 生成研究报告初稿
```
请基于知识库内容，生成关于 [研究主题] 的研究报告初稿：
1. 研究背景和意义
2. 核心概念和理论
3. 主要观点和发现
4. 实践应用和案例
5. 结论和展望
引用相关笔记作为支撑
```

## 🛠️ 维护管理类

### 18. 创建主题 MOC
```
[主题名] 下的笔记数量已经达到 5 篇，请创建主题地图 MOC - [主题名].md：
- 核心理论列表（带简介）
- 应用场景
- 相关主题链接
- 主题下的所有笔记索引
```

### 19. 检查并更新 MOC
```
请检查所有 MOC 文件是否需要更新：
1. 列出 03_Knowledge/ 下所有 MOC 文件
2. 检查每个 MOC 是否包含最新的笔记
3. 更新过时的链接和索引
4. 标记需要补充内容的领域
```

### 20. 清理重复内容
```
请帮我检查知识库中的重复内容：
1. 识别相似或重复的笔记
2. 标记需要合并的文件
3. 建议合并策略
4. 保留最完整的版本，删除重复内容
```

### 21. 优化标签系统
```
请帮我优化知识库的标签系统：
1. 统计所有标签的使用频率
2. 识别含义相同但名称不同的标签
3. 提出标签合并建议
4. 规范标签命名（扁平化、一致性）
5. 生成标签优化方案
```

### 22. 备份知识库
```
请帮我执行知识库备份：
1. 检查是否有未提交的更改
2. 创建 git commit，提交信息格式为"知识库备份: YYYY-MM-DD HH:MM:SS"
3. 推送到远程仓库（如果已配置）
4. 生成备份报告
```

### 23. 恢复知识库
```
请帮我从指定提交恢复知识库：
[提交哈希或时间点]
列出将要恢复的更改，确认后执行恢复操作
```

## 🎯 组合工作流

### 24. 每日知识管理流程
```
请执行每日知识管理流程：
1. 检查 00_Raw_Sources/ 中是否有新内容，如有则执行整理
2. 创建今日的每日笔记（如果还没创建）
3. 检查是否有主题笔记数量达到 5 篇，如有则创建 MOC
4. 快速检查是否有孤立笔记需要建立链接
5. 生成今日知识管理简报
```

### 25. 每周知识维护流程
```
请执行每周知识维护流程：
1. 整理 00_Raw_Sources/ 中的所有临时内容
2. 生成知识网络报告（孤立笔记、核心节点、标签统计）
3. 检查并更新所有 MOC 文件
4. 识别并清理重复内容
5. 提出知识库优化建议
6. 执行备份
```

### 26. 新知识学习流程
```
我要学习关于 [新主题] 的知识，请帮我：
1. 创建学习项目文档放入 04_Projects/
2. 从知识库中搜索相关背景知识
3. 生成学习计划和学习路径
4. 创建学习笔记模板
5. 设置学习里程碑和检查点
```

### 27. 项目知识沉淀流程
```
项目 [项目名称] 即将结束，请帮我沉淀项目知识：
1. 收集项目相关的所有笔记和文档
2. 创建项目总结文档
3. 提取项目中的最佳实践和经验教训
4. 将通用知识移动到 03_Knowledge/ 对应领域
5. 将项目文档移动到 04_Projects/ [项目名]/ 已完成/
6. 更新相关 MOC
```

## 🚀 高级操作

### 28. 知识库迁移准备
```
我要将知识库迁移到新环境，请帮我：
1. 生成知识库清单（文件统计、目录结构）
2. 检查所有内部链接是否有效
3. 识别可能丢失的关联
4. 生成迁移检查清单
5. 创建迁移后的验证方案
```

### 29. 知识库质量审计
```
请对知识库进行全面质量审计：
1. **内容质量**：检查是否有空文件、无意义文件
2. **结构质量**：检查目录组织是否合理
3. **关联质量**：检查孤立笔记、断链情况
4. **标签质量**：检查标签使用规范性
5. **命名质量**：检查文件命名规范性
6. 生成审计报告和改进建议
```

### 30. 知识库重构建议
```
请基于当前知识库状态，给出重构建议：
1. 分析现有知识结构的优缺点
2. 提出目录结构优化方案
3. 建议需要创建的 MOC
4. 识别需要拆分或合并的笔记
5. 提出标签系统改进方案
6. 制定重构实施计划
```

## 💡 使用提示

### 如何使用这些命令

1. **直接复制粘贴**：选择合适的命令，复制到 Claude 对话框中
2. **替换占位符**：将 `[方括号]` 中的内容替换为实际内容
3. **组合使用**：可以根据需要组合多个命令
4. **自定义调整**：根据实际情况调整命令内容

### 最佳实践

- **定期执行**：建议每周执行一次维护流程
- **及时整理**：新内容及时放入 00_Raw_Sources，定期整理
- **保持链接**：创建新笔记时，主动建立与已有笔记的链接
- **规范命名**：使用描述性的文件名，避免"未命名"等
- **标签精简**：每个笔记使用 3-5 个核心标签即可

### 快捷操作

- **快速记录**：使用命令 #2 记录灵感
- **快速搜索**：使用命令 #11 搜索主题
- **快速总结**：使用命令 #13 生成主题总结
- **快速备份**：使用命令 #22 执行备份
