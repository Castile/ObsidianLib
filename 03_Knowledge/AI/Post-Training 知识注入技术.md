# Post-Training 知识注入技术

Post-Training 阶段的核心是将领域知识可靠注入模型，实现模型对齐与知识固化。

## 两条主流技术路径

### 1. 指令合成

**原理**：将领域知识转化为高质量指令-响应对

**实现方式**：
- Self-Instruct：自动生成指令
- Evol-Instruct：指令演化
- 知识图谱引导的指令生成

**知识转化形式**：
- (问题-答案) 对
- (上下文-输出) 对

**目标**：让模型学会领域范式

### 2. 轨迹合成

**原理**：编码专家解决问题的完整过程

**包含内容**：
- 多步推理过程
- 工具调用序列
- API 执行链路
- 工作流 SOP

**目标**：让模型学会"怎么做"，而不只是"知道什么"

### 3. 知识蒸馏

**技术链路**：
```
强模型 LLM 生成伪标签
    ↓
构建监督数据集
    ↓
在小模型 SLM 上做微调
    ↓
蒸馏得到轻量化但能力接近的模型
```

## 知识形式的变化

### 传统 → 大模型时代

| 知识类型 | 传统形式 | 大模型时代形式 |
|----------|----------|----------------|
| 领域规则 | 专家规则库 | 指令数据中的隐式约束 |
| 实体关系 | 知识图谱三元组 | 参数化表示（隐空间分布式表示） |
| 流程知识 | 工作流引擎 | 轨迹数据、多步推理样本 |
| 质量标准 | 人工评估规则 | Reward Model、偏好数据、评估元数据 |

## 知识的可蒸馏性

**核心发现**：知识的可蒸馏性取决于其结构化程度

**高度形式化知识**（易蒸馏）：
- 代码
- 知识图谱
- 规则
- 表格
- 数据库记录

**非结构化知识**（难蒸馏）：
- 文本
- 语音
- 自然语言文档

**原因**：
- 形式化知识：结构清晰、语义明确
- 非结构化知识：模糊、噪声大、难以精准提取
