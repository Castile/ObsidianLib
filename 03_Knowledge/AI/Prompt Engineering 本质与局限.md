# Prompt Engineering 本质与局限

Prompt Engineering 是大模型时代知识工程的重要阶段，但存在明显的局限性。

## 本质：知识的即时载体

**核心定义**：将 Prompt 视为知识的即时载体，通过上下文学习 (In-context Learning) 实现能力的瞬时激活

**工作机制**：
- Prompt 一次性注入知识
- 模型通过 ICL 瞬时学习
- 无需修改模型参数

## 主要注入形式

### 1. Few-shot 示例 (ICL)

**特点**：相当于微型知识库
**用途**：
- 提高指令遵循能力
- 增强相关性任务表现
- 如分类任务

### 2. CoT 思维链

**特点**：推理策略知识的外化
**作用**：引导模型按逻辑步骤思考

### 3. System Prompt

**承载内容**：
- 角色定义
- 行为约束
- 输出规范等元知识

## 明显的局限性

### 1. 一次性与无状态特征

**问题**：
- Prompt 每次都需要重新构建
- 无法在对话间保持状态
- 知识无法持久化

### 2. 难以承载大规模结构化知识

**原因**：
- Token 长度限制
- 结构化知识需要大量空间
- 无法有效组织复杂知识体系

### 3. 模型性能高度敏感

**影响因素**：
- 知识的组织方式
- 顺序效应 (Order Effect)
- 注意力衰减 (Attention Decay)

**表现**：
- 同样的知识，不同组织方式效果差异巨大
- 长Prompt中前面的信息可能被忽略
- 知识在Prompt中的位置影响模型关注度

## 适用场景

✅ **适合**：
- 简单知识注入
- 临时性任务引导
- 小规模示例展示
- 快速原型验证

❌ **不适合**：
- 大规模知识库集成
- 复杂逻辑推理
- 长期知识存储
- 企业级应用
