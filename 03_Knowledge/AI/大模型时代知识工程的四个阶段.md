# 大模型时代知识工程的四个阶段

大模型时代的知识工程可细分为四个递进阶段，每个阶段解决不同层面的知识注入和利用问题。

## 四个阶段概览

### 1. Post-Training 阶段

**核心目标**：知识注入与模型微调
- 将领域知识固化到模型权重内部
- **技术手段**：
  - 微调 (Fine-tuning)
  - 持续预训练
  - 知识注入
  - LoRA/QLoRA 参数高效微调
  - 知识蒸馏

### 2. Prompt Engineering 阶段

**核心目标**：指令设计、上下文引导与推理控制
- 通过 Prompt 实现即时知识调用
- **技术手段**：
  - 指令设计
  - 思维链 (CoT)
  - Few-shot 学习
  - 结构化提示

### 3. Context Engineering 阶段

**核心目标**：外部知识库的动态检索与 RAG 集成
- 构建正确的上下文作为 LLM 的"内存管理系统"
- **技术手段**：
  - 外部知识库
  - 向量数据库
  - 混合检索
  - RAG、GraphRAG

### 4. Harness Engineering 阶段 (2026年核心范式)

**核心目标**：系统级约束、工具集成与动态执行
- 解决"系统如何安全、稳定、可重复地运行"问题
- **技术手段**：
  - 工具调用
  - 状态管理
  - 权限沙箱
  - 多 Agent 编排
  - 自检护栏
  - 系统级 Harness

## 阶段演进逻辑

```
Post-Training    →  Prompt Engineering  →  Context Engineering  →  Harness Engineering
(知识固化)          (即时调用)            (内存管理)              (系统约束)
```

## 核心问题解决

- **Post-Training**：解决"知识如何存入模型"
- **Prompt Engineering**：解决"如何让模型调用知识"
- **Context Engineering**：解决"如何管理外部知识"
- **Harness Engineering**：解决"如何让系统稳定运行"
