# Harness Engineering 系统级约束

Harness Engineering 是2026年知识工程最深刻的范式升级，解决"系统如何安全、稳定、可重复地运行"的核心问题。

## 核心定位

如果说：
- **Prompt** 解决"问什么"
- **Context** 解决"给什么"
- **Harness** 解决"系统如何安全、稳定、可重复地运行"

## 多维视角理解

### 系统架构视角

```
模型 = 引擎
上下文 = 燃料
Harness = 整车（方向盘、刹车、仪表盘、安全气囊、监控系统）
```

### 核心逻辑视角

**知识不再只是"模型知道什么"，而是"系统如何约束、验证、编排模型的行为"**

### 组成要素视角

- **状态管理**：维护系统状态
- **权限沙箱**：控制操作权限
- **重试机制**：处理失败情况
- **检查点**：支持恢复和回滚
- **评审循环**：持续质量保证

### 载体形式视角

通过配置文件定义：
- `AGENTS.md`
- `CLAUDE.md`

**定义内容**：
- 模块边界
- 不变量
- Linter 规则
- 权限沙箱

## 六类知识类型

### 1. 架构知识

**内容**：模块边界、依赖关系、不变量
**载体**：AGENTS.md、CLAUDE.md

### 2. 验证知识

**内容**：什么算正确
**载体**：
- 测试用例
- Lint 规则
- CI/CD 流程
- 质量门

### 3. 权限知识

**内容**：控制 Agent 能做什么、不能做什么
**载体**：
- 沙箱
- 能力白名单
- 操作禁区
- 数据隔离

### 4. 恢复知识

**内容**：失败时如何回退
**载体**：
- 状态机
- 回滚策略
- 超时机制
- 异常捕获
- 检查点策略

### 5. 编排知识

**内容**：多 Agent 协作
**载体**：
- 工作流定义
- Review Loop
- 调用协议
- 角色分工

### 6. 演化知识

**内容**：系统自我迭代
**载体**：
- 技能文件
- 自动更新机制
- 从错误日志学习
- 自动加固护栏
- 自迭代配置

## 工业级可靠系统的实现

正是这些要素的加持，使得 **Agent 这类工业级可靠系统的实现成为可能**。

### 关键能力

- **稳定性**：通过状态管理和恢复机制
- **安全性**：通过权限沙箱和验证知识
- **可控性**：通过架构知识和约束规则
- **可扩展性**：通过编排知识和演化机制

## 实践意义

Harness Engineering 将知识工程从"如何让模型知道知识"提升到"如何让系统可靠地使用知识"，是 AI 系统从实验走向生产的关键。
